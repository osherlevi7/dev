Cloud Armor

Cloud Armor is a security service provided by Google Cloud Platform (GCP) that helps protect websites and applications from DDoS attacks and other web-based threats. It is a managed service that provides both global load balancing and security policies for web applications hosted on GCP.

Cloud Armor uses a set of pre-configured security policies, called security rules, that are used to block malicious traffic and allow legitimate traffic. These security rules are based on IP addresses, geographic locations, and request patterns, and can be customized to meet the specific security requirements of a web application.

Cloud Armor integrates with other GCP services, such as Google Kubernetes Engine (GKE), Compute Engine, and Cloud Load Balancing, to provide a comprehensive security solution for web applications hosted on GCP. Cloud Armor also provides logging and monitoring capabilities that enable administrators to monitor traffic and security events in real-time, and to create custom alerts based on specific security events or traffic patterns.

Key Components

  • Security Rules: Cloud Armor provides a set of pre-configured security rules that can be customized to meet the specific security requirements of a web application. These rules are based on IP addresses, geographic locations, and request patterns.
  • IP Lists: Cloud Armor allows administrators to create custom IP lists to block or allow traffic from specific IP addresses or ranges.
  • WAF Rules: Cloud Armor supports the use of Web Application Firewall (WAF) rules, which are used to block malicious traffic and protect web applications from common attacks such as SQL injection and cross-site scripting (XSS).
  • DDoS Protection: Cloud Armor provides DDoS protection by using Google's global infrastructure to absorb and mitigate DDoS attacks before they reach the web application.
  • Logging and Monitoring: Cloud Armor provides logging and monitoring capabilities that enable administrators to monitor traffic and security events in real-time, and to create custom alerts based on specific security events or traffic patterns.
  • Integration with Google Cloud Load Balancing or any: Cloud Armor integrates with Google Cloud Load Balancing to provide a comprehensive security solution for web applications hosted on GCP. Overall, these components work together to provide a comprehensive security solution for web applications hosted on GCP.

Architecture

Identity-Aware Proxy (IAP) verifies a user's identity and then determines whether that user should be permitted to access an application. To enable IAP for the external HTTP(S) load balancer, you enable it on the load balancer's backend services. Similarly, edge Google Cloud Armor security policies are attached to the backend services of an external HTTP(S) load balancer. If Google Cloud Armor security policies and IAP are both enabled for a backend service of an external HTTP(S) load balancer, the IAP evaluation happens first. If IAP blocks a request, Google Cloud Armor does not evaluate the request. If IAP successfully authenticates a request, Google Cloud Armor then evaluates the request. The request is blocked if a Google Cloud Armor security policy produces a deny decision.

Google Cloud Platform

Google Cloud Armor helps protect your infrastructure and applications from Layer 3/Layer 4 network or protocol-based volumetric distributed denial-of-service (DDoS) attacks, volumetric Layer 7 attacks, and other targeted application attacks. It leverages Google's global network and distributed infrastructure to detect and absorb attacks and filter traffic through user-configurable security policies at the edge of Google's network, far upstream of your workloads.
